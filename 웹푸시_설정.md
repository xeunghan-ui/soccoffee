# 웹 푸시 알림 설정 가이드 (2026-07)

멤버앱에서 "알림 받기"를 켜면 공지·세션 리마인드가 폰 알림으로 갑니다.
스택: 표준 Web Push(VAPID) + Supabase(구독 저장) + GitHub Actions(발송, 공개 저장소라 무료).

## 1회 설정 (총괄이 해야 할 일 — 약 5분)

### ① Supabase 테이블 생성
Supabase 대시보드 → SQL Editor에서 실행:

```sql
create table if not exists push_subs (
  endpoint text primary key,
  data jsonb not null,
  member_id bigint,
  member_name text,
  created_at timestamptz default now()
);
alter table push_subs enable row level security;
create policy "open all" on push_subs for all using (true) with check (true);
```

### ③ 푸시 후 확인
`.github/workflows/push-notify.yml`이 매시간 10분에 자동 실행됩니다.
(참고: `scripts/push-notify.yml.move-to-github-workflows`는 같은 내용의 백업 — 이미 워크플로가 있으면 지워도 됨)
GitHub → Actions 탭에서 `push-notify` 실행 로그 확인.

## 발송 트리거 (매시간 10분에 확인)
콘텐츠형(등록 후 최대 1시간 내 발송):
- 📣 **새 공지** 게시 → 전체
- 🚗 **새 카풀** 게시 → 전체
- 📅 **새 세션 일정** 게시 → 전체

날짜형(저녁 18~21시 KST에 1회):
- ⚽ **내일 세션 리마인드** → 전체
- ⏰ **참석 마감 하루 전** → **미응답·미정인 멤버만** (타겟)
- 🗳️ **투표 시작** (매월 25일) → 전체
- 💰 **회비 납부 시작** (매월 15일) → 전체
- 💸 **회비 마감(25일) 하루 전**(24일) → **미납 멤버만** (타겟)

수동 발송: GitHub → Actions → push-notify → Run workflow → 제목/내용 입력 (폰 GitHub 앱에서도 가능)
중복 방지 상태는 `club_settings` `id='push_state'` 행에 저장 (이 행은 Actions만 씀).
최초 1회 실행 때는 기존 공지·카풀·세션을 기록만 하고 알리지 않아요(설치 직후 알림 폭탄 방지).
타겟 발송은 구독 시 저장된 member_id 기준 — 로그인한 상태에서 알림을 켠 멤버에게만 정확히 갑니다.

## 멤버 안내 (카톡 공지용)
> 📱 싸커피 앱에서 알림을 받아보세요!
> 1. 사이트 접속 → 더보기 → 설정 → "알림 받기"
> 2. 아이폰은 먼저 [공유 → 홈 화면에 추가] 후 홈 화면 아이콘으로 열어서 켜기
> 공지·세션 전날 리마인드가 폰으로 옵니다.

## 참고
- 아이폰: iOS 16.4+ 에서 홈 화면 추가(PWA)로 연 경우에만 푸시 가능 — 사파리로 그냥 열면 안 됨
- 안드로이드/PC 크롬: 사이트에서 바로 가능
- 구독 취소: 더보기 → 설정 → "알림 켜짐" 눌러 끄기 (만료된 구독은 발송 시 자동 정리)
