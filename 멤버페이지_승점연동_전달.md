# 멤버 페이지 전달: 팀 승점 연동 (팀빌더 → 멤버)

팀빌더에서 매치 결과 기록이 준비됐습니다. 멤버 페이지의 "팀 승률"을 **승점제**로 교체·연결하면 됩니다. 팀빌더 쪽 데이터 공급은 **완료** 상태입니다.

## 1. 데이터 위치
- `club_settings` 테이블, `id='teambuilder'` 행의 `data.matches` 배열.
- 멤버 페이지는 이미 이 teambuilder 데이터를 읽고 있으므로(players/sessions), `data.matches`만 추가로 읽으면 됩니다.

## 2. matches[] 스키마 (현재/신포맷)
```json
{
  "id": "1782...",
  "date": "2026-06-24",
  "blackScore": 14,
  "whiteScore": 9,
  "black": [35, 39, 21, ...],      // 블랙팀 선수 id 배열 (= 팀 배분 teamA)
  "white": [18, 8, 38, ...],       // 화이트팀 선수 id 배열 (= teamB)
  "blackNames": ["김균원", ...],    // 참고용 이름(표시용)
  "whiteNames": ["김두은", ...]
}
```
- **선수 매칭은 `black`/`white`의 id로** 하세요(이름 아님 — 동명이인 위험).
- `potm` 키가 있을 수 있으나 **무시**하세요(구버전 입력 잔재, 값 없음).

## 3. 승점 규칙 (확정)
- **승 3점 / 무 1점 / 패 0점.**
- **그날 종합**으로 승패 판정. 스코어가 단일이면 그대로(`blackScore` vs `whiteScore`).
- **승점은 배분 명단(black/white) 전원에게 부여** — 그날 참석 여부와 무관. 불참자도 소속 팀의 결과를 그대로 공유. (세션 attendees와 교집합 내지 **않음**.)
- 선수 개인 승점 = 본인이 속했던 팀이 받은 승점의 **누적 합**.

## 4. 계산 로직 (의사코드)
```js
// matches: data.matches
const pts = {}; // playerId -> 누적 승점
for (const m of matches) {
  const bs = m.blackScore, ws = m.whiteScore;
  const blackPts = bs > ws ? 3 : bs === ws ? 1 : 0;
  const whitePts = ws > bs ? 3 : ws === bs ? 1 : 0;
  (m.black || []).forEach(id => pts[id] = (pts[id] || 0) + blackPts);
  (m.white || []).forEach(id => pts[id] = (pts[id] || 0) + whitePts);
}
// pts[playerId] = 그 선수의 총 승점
```

## 5. 현재 실데이터 (예시 검증용)
- 매치 1건: **2026-06-24, 블랙 14 : 화이트 9 → 블랙 승.**
- 결과: 블랙팀 18명 각 +3점, 화이트팀 18명 각 0점.
- (블랙에 정하림 포함 — 그날 불참이지만 규칙상 승점 부여됨. 의도된 동작.)

## 6. 구포맷 호환 (참고)
- 과거 기록 중 일부는 `gameA`/`gameB`(전·후반 두 게임) + `blackTeamA/B`(이름 기반) 형태일 수 있음.
- 그런 항목은 **그날 종합 = 두 게임 스코어 합산**으로 승패 판정, 명단은 이름 기반이라 id 매칭 시 주의.
- 단, 현재 실제 데이터는 신포맷 1건뿐이라 당장은 신포맷만 처리해도 무방.

## 7. 표시
- "팀 승률" 문구/계산을 위 승점 합으로 교체. 랭킹·개인 카드 등 표시 위치는 멤버 페이지 재량.
