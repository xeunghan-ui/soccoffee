## 2. 배포 방법

1. 코드 수정은 로컬(`~/Documents/Claude/Projects/socoffee`)에서.
2. 커밋 후 **푸시**:
   ```
   cd ~/Documents/Claude/Projects/socoffee && git push origin main
   ```
3. GitHub가 자동 빌드/배포(Actions의 "pages build and deployment"). 보통 1~5분.
4. **중요 — 캐시**: GitHub Pages는 HTML을 CDN에 캐시해서 최신 변경이 바로 안 보일 수 있어요. 확인할 땐 주소 끝에 `?v=숫자`를 붙여 캐시를 우회(예: `…/member.html?v=2`). 그래도 안 바뀌면 Actions 탭에서 빌드 성공(초록)인지 확인.
5. 저장소는 **public 유지**해야 함(무료 플랜에서 private면 Pages가 꺼짐).

## 3. Supabase (데이터베이스)

- 프로젝트 URL: `https://fjgxhguogsuypcdzcieg.supabase.co`
- 클라이언트엔 **공개용 anon(publishable) 키만** 사용. `service_role` 키는 절대 코드에 넣지 말 것.
- 테이블 생성 SQL: 저장소의 **`Supabase_전체설정.md`** (재실행 안전). 새 환경이면 이거 한 번 실행.

### 테이블
| 테이블 | 내용 |
|---|---|
| `club_settings` | jsonb 보관함. `id='current'`(사이트 설정·세션·팀구분·**PIN**·명단), `id='teambuilder'`(팀빌더 전체 데이터·휴면) |
| `notices` | 공지. 컬럼: title, body, pinned, **publish_at(노출 시작), hide_at(노출 종료)**, created_at |
| `attendance` | 참석. session_id, member_id, status(yes/no/maybe) |
| `dues` | 회비. month, member_id, paid, amount |
| `potm_votes` | 투표(이달의 선수/가장 성장). month, category(mvp/growth), voter_id, candidate_id |
| `rides` | 카풀 |

- **RLS(보안)**: 모든 테이블이 "익명 키로 누구나 읽기/쓰기"(링크 기반 신뢰 그룹용, casual 보안). 즉 소스를 본 사람은 데이터 읽기/수정/삭제가 기술적으로 가능. 동호회 수준에서 수용 중. 더 강화하려면 RLS를 손봐야 함(무로그인 구조 변경 필요).

## 4. 로그인 / 권한

- **로그인 = 이름 선택 + PIN 4자리**. (등번호 로그인은 폐지 — 등번호가 소개 페이지에 공개돼 취약해서)
- PIN은 **SHA-256 해시**로만 `club_settings.current.pins`에 저장(평문 미저장). 최초 로그인 시 등록.
- **운영진(ADMIN)**: 이름이 `ADMIN_NAMES = ['홍순인','박승한','원재식','최승호','정희범']` 에 있으면 운영진. 자기 PIN으로 보호됨.
- 운영진 → **설정** 탭에서 멤버 **PIN 초기화**(잊었을 때) + 설정 여부 확인 가능.

## 5. 멤버 앱(member.html) 기능별 메모

- **공지**: 운영진이 작성/수정/삭제. 본문 URL **링크 자동 연결**. **노출 기간(시작~종료)** 설정 가능 — 기간 밖이면 멤버에게 자동 숨김(운영진 목록엔 상태 뱃지).
- **참석**: 세션별 참석/불참/미정. **신청 마감 = 매치일 직전 일요일(전주 일요일)** 자동(세션마다 변경 가능). 마감 후 **일반 멤버 차단, 운영진은 변경 가능**. 운영진은 **명단 현황에서 다른 멤버 참석도 직접 변경**(상태 뱃지 클릭, 순환). 명단은 응답 상태별로 그룹 표시.
- **회비**: **매월 25일부터 다음 달** 회비를 표시(`duesMonth()`). **휴면자 제외**. **미납 명단** 별도 노출. 운영진이 이름 눌러 납부/미납 토글(스크롤 위치 유지).
- **카풀**(탭 이름 '카풀', 내부적으로 list): 카풀 모집/탑승.
- **투표**: 이달의 선수(MVP) + 가장 성장한 선수. 부문별 1인 1표.
- **랭킹**: 출석/모범 등 랭킹(연도 기본 2026).
- **운영진**: 공지·세션·회비·투표·**설정**(맨 끝, 팀구분 토글 + PIN 관리). 세션은 "＋ 세션 추가" 버튼으로 폼 열림. 세션에 **세부내용(desc)** 넣으면 소개 페이지 MATCH DAY에서 그 줄 클릭 시 펼쳐짐.

- **휴면**: 월 단위(예: '2026-07'). 사이트의 회비·참석·홈 명단에서 해당 월 휴면자 자동 제외.
- **"🔄 사이트 출석"**: 사이트 참석(yes) 기록을 팀빌더로 가져옴(이름 매칭, 기존 날짜 스킵).

## 7. 자주 하는 작업

### 갤러리 사진 추가
1. 사진을 `img/` 폴더에 넣음(원본은 보통 10~20MB).
2. **반드시 압축**(안 하면 페이지 느려짐). Python(PIL):
   ```python
   from PIL import Image, ImageOps
   im = ImageOps.exif_transpose(Image.open("img/파일.jpg")).convert("RGB")
   im.thumbnail((1920,1920))
   im.save("img/파일.jpg","JPEG",quality=82,optimize=True,progressive=True)
   ```
   (회전 정보 보정 위해 `exif_transpose` 꼭.)
3. `index.html`의 `.gallery` 안에 `<img src="img/파일.jpg" alt="싸커피 활동" loading="lazy">` 추가.
4. 갤러리는 그리드 메이슨리(비율 유지) + dense 패킹. 초기 10장 노출, "더보기/접기" 토글.

### 대표(공유 미리보기) 이미지
`index.html`의 `og:image` / `twitter:image` 메타 태그 수정.

### 코드 수정 후 검증
HTML 인라인 스크립트는 문법 깨지기 쉬우니, 커밋 전 `node --check`로 확인하는 습관.
